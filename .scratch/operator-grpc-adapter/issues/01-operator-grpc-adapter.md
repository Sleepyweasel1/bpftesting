# Add hold-operator gRPC adapter for CapturelistService

## What to build

Add a generated tonic gRPC client to `hold-operator` and wrap it in a concrete `HoldPacketClient` adapter struct that exposes typed methods: `add_rule(IpAddr)`, `remove_rule(IpAddr)`, `list_rules() -> Vec<IpAddr>`, `replay_rule(id: u64)`. The adapter is constructed from a daemon endpoint URL and stored in `OperatorContext`. Introduce a trait `CaptureControl` backed by the adapter so tests can substitute a fake.

End-to-end: `hold-operator` binary starts, connects to `hold-packet` gRPC endpoint, calls `add_rule("10.96.0.1")`, and receives a success response — verifiable via the existing gRPC server run in an integration test.

- [ ] `hold-operator/Cargo.toml` references the `holdpacket.proto` build step and tonic client code is generated at compile time.
- [ ] `HoldPacketClient` wraps the generated stub and compiles against the current proto definition.
- [ ] `CaptureControl` trait is defined; `HoldPacketClient` and a `FakeCaptureControl` test double both implement it.
- [ ] A unit test using `FakeCaptureControl` verifies that the adapter correctly translates `IpAddr` arguments to proto string form.
- [ ] `OperatorContext` carries an `Arc<dyn CaptureControl>` field.

## Agent Brief

**Category:** enhancement
**Summary:** Introduce `CaptureControl` trait and `HoldPacketClient` adapter in `hold-operator`, backed by the generated tonic gRPC client, so the reconciler can call `hold-packet` through a mockable interface.

**Current behavior:**
`hold-operator` already has tonic and the proto build wired up and the generated `CapturelistServiceClient` is imported in `main.rs`. However, gRPC calls are made as raw inline tonic calls scattered through `execute_agent_intent`, with no typed wrapper and no test seam. `OperatorContext` holds only a raw `hold_packet_grpc_addr: String`.

**Desired behavior:**
- A `CaptureControl` async trait defines typed operations: `add_rule(IpAddr) -> Result<()>`, `remove_rule(IpAddr) -> Result<()>`, `list_rules() -> Result<Vec<IpAddr>>`, `replay_rule(id: u64) -> Result<()>`. Error type should be compatible with the reconciler's error handling.
- `HoldPacketClient` is a struct that implements `CaptureControl` by constructing a tonic channel from its endpoint URL, calling the generated `CapturelistServiceClient`, and converting proto string IPs to/from `IpAddr`.
- `FakeCaptureControl` is an in-process test double implementing `CaptureControl`. It records calls (for assertion) and returns configurable results.
- `OperatorContext` carries an `Arc<dyn CaptureControl + Send + Sync>` instead of the raw addr string. It is constructed with a real `HoldPacketClient` in `main()`.
- Existing call sites in `execute_agent_intent` (or wherever gRPC is currently called) are refactored to go through the `CaptureControl` trait.
- A unit test exercises `FakeCaptureControl` to confirm that the adapter correctly round-trips `IpAddr` ↔ dotted-decimal string (the proto wire format).

**Key interfaces:**
- `CaptureControl` trait — new; async methods returning `Result`. Must be object-safe or used via `Arc<dyn …>` with `async_trait` or equivalent.
- `HoldPacketClient` struct — new; constructed from a `String` endpoint URL; implements `CaptureControl`.
- `FakeCaptureControl` struct — new; for use in `#[cfg(test)]` blocks; records which methods were called and with what args.
- `OperatorContext.hold_packet_grpc_addr: String` — replace with a `capture_control: Arc<dyn CaptureControl + Send + Sync>` field.
- `execute_agent_intent` (or equivalent) — remove direct tonic call; delegate through the `CaptureControl` field on `ctx`.

**Acceptance criteria:**
- [ ] Proto build step and generated client code exists in `hold-operator` (already present — verify it still compiles after the refactor).
- [ ] `HoldPacketClient` compiles against the current `holdpacket.proto` definition with all four RPCs (`AddRule`, `RemoveRule`, `ListRules`, `ReplayRule`).
- [ ] `CaptureControl` trait is defined and `HoldPacketClient` implements it.
- [ ] `FakeCaptureControl` test double implements `CaptureControl`.
- [ ] A unit test using `FakeCaptureControl` asserts that `add_rule(IpAddr::V4("10.96.0.1".parse().unwrap()))` causes a call with ip string `"10.96.0.1"`.
- [ ] `OperatorContext` carries `Arc<dyn CaptureControl + Send + Sync>` and compiles.
- [ ] `cargo test -p hold-operator` passes.

**Out of scope:**
