## Agent Brief

**Category:** enhancement

**Summary:** Harden degraded idle-decision behavior so transport failures, stale snapshots, and partial/ambiguous idle evidence consistently produce safe `RecoverableFailure` outcomes with actionable diagnostics.

**Current behavior:**
The reconcile path already reads daemon idle-status snapshots and reports explicit failures when traffic signal reads fail or when all target IPs are unknown/not-captured. It also requeues on recoverable outcomes. Existing tests cover some degraded paths, but behavior for partially unknown snapshots and other ambiguous evidence cases is not fully pinned as a deterministic failure contract.

**Desired behavior:**
Idle-decision failure semantics should be explicit and safe:
- Transport/unavailable reads -> `RecoverableFailure` with clear stale/transport reason.
- Stale/invalid evidence (including missing required target coverage) -> `RecoverableFailure` with actionable message.
- Partial-data/ambiguous snapshots must not silently permit unsafe scale transitions.

Repeated degraded reconciles must remain idempotent and avoid mutating deployment scale or capture state based on uncertain evidence.

**Key interfaces:**
- `read_traffic_signal(...)`: centralize and tighten degraded-snapshot classification (stale, absent, partial/ambiguous).
- Reconcile assessment result assembly (`assess_reconcile_with_readers`): ensure failure reasons map deterministically to `RecoverableFailure` and message text.
- `ScaleToZero` status condition construction: preserve `Reconciled=False`, `reason=RecoverableFailure`, and stable message semantics for degraded paths.
- Test doubles (`FakeCaptureControl`, fake kube state reader): drive transport, stale, partial, and repeated-failure scenarios.

**Acceptance criteria:**
- [ ] Reconcile maps idle-status transport failures to deterministic `RecoverableFailure` outcomes with clear message text.
- [ ] Reconcile maps stale/absent/partial idle evidence to deterministic `RecoverableFailure` outcomes instead of unsafe scale transitions.
- [ ] Under repeated degraded reads, reconcile remains idempotent and does not apply unsafe scale mutations.
- [ ] Condition/reason/message contract is verified for degraded paths (`Reconciled=False`, `RecoverableFailure`, actionable message).
- [ ] Unit/integration tests cover degraded scenarios including transport error, all-unknown, partial unknown coverage, and repeated retries.

**Out of scope:**
- New daemon APIs or idle-algorithm redesign.
- Policy changes unrelated to degraded-path safety/diagnostics.
- Broader autoscaling behavior beyond idle-decision failure semantics.
