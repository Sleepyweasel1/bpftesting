# Reconciler Scale Action Path (Patch Deployment Scale)

## What to build

Add an idempotent scale-action path in hold-operator so reconcile can patch target Deployment scale up/down when a decision requires it. This slice should include safe retry behavior, status outcomes, and required RBAC updates.

End-to-end: reconcile computes desired replica action -> operator patches Deployment scale subresource -> status reflects succeeded/no-op/recoverable-failure.

## Acceptance criteria

- [ ] hold-operator has a tested API seam for get/patch Deployment scale in target namespace.
- [ ] Reconcile can execute explicit scale-to-0 and scale-to-1 actions idempotently with retryable error handling.
- [ ] Kubernetes manifests/RBAC grant only the minimum permissions required for Deployment scale reads/writes.

## Agent Brief

**Category:** enhancement

**Summary:** Add an idempotent Deployment scale-action capability to the Reconciler so it can patch target Deployment replicas up/down when directed by ScaleToZero intent, with safe retry behavior and observable status outcomes.

**Current behavior:**
Reconcile computes a `ReconcileIntent` (HoldCapture, RemoveCapture) based on the target Deployment's current replica count. It then drives the hold-packet daemon's capture state. However, it does not patch the target Deployment's scale subresource. In a full traffic-driven scale-to-zero loop, the Reconciler must be able to trigger the actual scale event once replay is complete.

**Desired behavior:**
Reconcile should be able to execute explicit scale-to-0 and scale-to-1 actions idempotently. Scale operations must be retryable with backoff (following `AGENT_RETRY_BACKOFFS` pattern), and outcomes must be recorded in `ScaleToZeroStatus` with clear intent/outcome/message semantics distinguishing NoOp, Succeeded, and RecoverableFailure.

**Key interfaces:**
- `KubeStateReader` trait — extend with `patch_deployment_scale(namespace, name, replicas)` method for an idempotent scale seam.
- `ReconcileIntent` — already defined as HoldCapture/RemoveCapture/Replay; consider whether a new Scale intent value is needed or if scaling is implicit in another flow.
- `ScaleToZeroStatus` — records outcome of scale operations via existing intent/outcome/message fields.
- `execute_agent_intent_for_ips()` — current control-plane mutation point; may need extension or a sibling `execute_scale_intent()` fn if scale actions are separate from capture actions.

**Codebase summary:**
- [main.rs](hold-packet/hold-operator/src/main.rs) — Reconciler loop reads ScaleToZero, calls `assess_reconcile()` to decide intent, executes intent via `execute_agent_intent_for_ips()`, records status.
- [rbac.yaml](hold-packet/deploy/vind/rbac.yaml) — currently grants deployments get/list/watch but not deployments/scale patch/update. Must be updated to grant minimum required permissions.
- [ADR-0005](docs/adr/0005-grpc-operator-control-plane.md) — gRPC is operator→daemon seam; scale operations are operator→Kubernetes seam (separate).
