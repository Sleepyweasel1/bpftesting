## Design Decisions

### Infrastructure-first approach
Scale action methods and `execute_scale_action()` are available and tested, but NOT YET wired into the main reconcile loop. This allows issues #03 (idle-driven scale-down) and #04 (wake-driven scale-up) to plug in their own decision logic without circular dependencies.

### Idempotency strategy
Handles the case where patch fails due to conflict but the desired state is already reached (via get_deployment_replicas check). No special error markers needed — if current == desired, it's a success.

### Error classification
Follows existing pattern — retryable errors get backoff, non-retryable errors return immediately.
