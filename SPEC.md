# Hold-Packet Domain Specification

## Purpose

hold-packet is an eBPF-based packet capture and replay system that enables scale-to-zero for Kubernetes services. When a service scales down to zero replicas, incoming network packets are intercepted by the eBPF TC classifier and redirected to a TAP device. A userspace daemon stages those packets and exposes a gRPC API. A Kubernetes operator (hold-operator) watches a `ScaleToZero` custom resource and drives the control loop: it tells hold-packet which IPs to capture and when to replay staged packets to wake a scaled-down service.

## Components

| Component | Crate | Role |
|---|---|---|
| eBPF classifier | `hold-packet-ebpf` | Kernel: intercept / redirect packets |

| Userspace daemon | `hold-packet` | Stage packets, expose gRPC, run idle monitor |
| Kubernetes operator | `hold-operator` | Watch CRDs, reconcile scale-to-zero lifecycle |

## eBPF Program

**Hook**: TC ingress classifier on the monitored interface (default `eth0`).

**Maps**

| Map | Type | Key | Value | Capacity |
|---|---|---|---|---|
| `STATEV4` | `HashMap` | `u32` (IPv4 destination) | `StateEntry` | 1 024 |
| `STATEV6` | `HashMap` | `u128` (IPv6 destination) | `StateEntry` | 1 024 |
| `TAP_IFINDEX` | `Array` | `u32` (index 0) | `u32` (ifindex) | 1 |

**`StateEntry`** (shared via `hold-packet-common`)

```rust
#[repr(C)]
pub struct StateEntry {
    pub last_seen_ns: u64,   // CLOCK_MONOTONIC timestamp of last seen packet
    pub packet_count: u64,   // total packets observed for this destination
    pub replay: u8,          // 0 = pass-through; 1 = redirect to TAP
}
```

**Classifier logic** (`hold_packet` function)

1. Parse Ethernet → IPv4 or IPv6 header to extract destination address.
2. Lookup destination in `STATEV4` / `STATEV6`.
3. If no entry → `TC_ACT_NEXT` (not a tracked IP, pass through).
4. If entry found:
   - Update `last_seen_ns` to current monotonic time.
   - Increment `packet_count`.
   - If `replay == 1` → `bpf_redirect(TAP_IFINDEX[0], 0)` (service scaled-to-zero).
   - If `replay == 0` → `TC_ACT_PIPE` (service active, pass through).

## Userspace Daemon (`hold-packet`)

### Configuration

| Flag | Default | Description |
|---|---|---|
| `--iface` | `eth0` | Interface to attach TC classifier |
| `--grpc-addr` | `[::]:50051` | gRPC listen address |
| `--idle-timeout-secs` | `300` | Seconds of inactivity before marking `replay=1` |

### Startup sequence

1. Load compiled eBPF object (embedded via `OUT_DIR`).
2. Attach `hold_packet` TC classifier to `--iface` ingress.
3. Create TAP device `tap1`; write its ifindex to `TAP_IFINDEX[0]`.
4. Wrap `STATEV4` / `STATEV6` map handles in `Arc<RwLock<>>`.
5. Spawn background tasks (see below).
6. Start gRPC server.

### Background tasks

| Task | Interval | Responsibility |
|---|---|---|
| **Idle monitor** | 30 s | Read all map entries; set `replay=1` if `last_seen_ns` is older than `idle_timeout_secs`; clear to `replay=0` if traffic resumed. |
| **Replayer runner** | Continuous | Read L2 frames arriving on TAP, parse src/dst IP, assign auto-incremented ID, store in `staged` map. |
| **Packet pruner** | 30 s | Evict staged packets older than 120 s (`STAGED_TTL`). |

### Packet staging

```rust
pub struct StagedPacket {
    pub data: Vec<u8>,        // raw L2 frame
    pub src_ip: IpAddr,
    pub dst_ip: IpAddr,
    pub len: usize,
    pub staged_at: Instant,
}

// Stored as: Arc<Mutex<HashMap<u64, StagedPacket>>>
// Key generated by: Arc<AtomicU64> (monotonically increasing)
```

Replay sends the raw L2 frame back through the TAP writer. The kernel processes it as an RX frame on `tap1`; under Option A, recapture prevention is structural because the classifier is attached to `eth0` ingress.

## gRPC API

**Proto package**: `holdpacket`  
**Service**: `CapturelistService` on port `50051`

| RPC | Request | Response | Semantics |
|---|---|---|---|
| `AddRule` | `{ ip: string }` | `{ success, error }` | Insert IP into capture maps with `replay=0`. |
| `RemoveRule` | `{ ip: string }` | `{ success, error }` | Delete IP from capture maps. |
| `ListRules` | `{}` | `{ ips: []string }` | Enumerate all tracked IPs (IPv4 + IPv6). |
| `ReplayRule` | `{ id: uint64 }` | `{ success, error }` | Look up staged packet by ID; write to TAP. |

IP strings are standard dotted-decimal IPv4 or colon-hex IPv6.

## Kubernetes Operator (`hold-operator`)

### Custom Resource

```
group:   scale.sleepy.com
version: v0
kind:    ScaleToZero
scope:   Namespaced
```

**Spec**

```rust
pub struct ScaleToZeroSpec {
    pub target_ref: TargetRef,   // { deployment_name, namespace }
}
```

**Status**

```rust
pub struct ScaleToZeroStatus {
    pub replicas: i32,
    pub checksum: String,
    pub last_updated: Option<Time>,
}
```

### Reconciliation contract

The controller watches `ScaleToZero` resources. For each resource the intended reconciliation loop is:

1. Fetch the target `Deployment` replica count.
2. If replicas == 0 (scaled down): call `AddRule(service_ip)` on the hold-packet gRPC service.
3. If a packet is staged for a captured IP and a wake-up trigger fires: scale the Deployment back up, then call `ReplayRule(id)` to deliver the held packet.
4. If replicas > 0 (service active): call `RemoveRule(service_ip)` so traffic flows normally.
5. Update `ScaleToZeroStatus` with current replica count and checksum.

> **Current implementation state**: The reconciler runs a starter loop (`Action::await_change()`) — the CRD watch and controller scaffolding are complete but state-mutation logic is not yet implemented.

## Data Flow

```
Inbound packet on eth0
        │
        ▼
  eBPF TC ingress classifier
  ├── Not in STATEV4/STATEV6 ──────────────────────────► TC_ACT_NEXT (pass)
  └── In map
       ├── replay=0 ────────────────────────────────────► TC_ACT_PIPE (pass)
       └── replay=1
            │
            ▼
       bpf_redirect(tap1)
            │
            ▼
       Replayer runner reads L2 frame from TAP
            │
            ▼
       StagedPacket stored → staged[id]
            │
            ▼
       Operator calls ReplayRule(id) via gRPC
            │
            ▼
       L2 frame written back to TAP
            │
            ▼
       Kernel delivers to service (no recapture)
```

## Invariants & Constraints

- A `StateEntry` with `replay=0` means the service is considered live; packets pass through.
- A `StateEntry` with `replay=1` means the service is scaled-to-zero; all new packets are redirected.
- The idle monitor is the only writer that flips `replay`; the operator drives IP membership via gRPC.
- Staged packets are ephemeral: TTL is 120 s; the operator must replay before eviction.
- Recapture prevention is a topology invariant for current architecture: TC classifier attached on `eth0` ingress and replay injected via TAP RX.
- Correctness boundary is delivery semantics, not per-interface policy equivalence.
- For historical context and decision rationale, see ADR-0007.
- Map capacity is 1 024 entries per address family; operator must not exceed this.
- IPv4 keys are stored as raw `u32` (network byte order); IPv6 keys as raw `u128`.
